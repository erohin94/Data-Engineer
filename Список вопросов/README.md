**2) Отличие DROP, DELETE, TRUNCATE?**

DELETE удаляет (частично или полностью) содержимое таблицы, после удаления всех строк посредством инструкции delete таблица продолжает существовать в базе данных. В Impala DELETE нет. 
В Impala DELETE поддерживается только для Kudu-таблиц.

DROP полностью удаляет таблицу: данные; структуру; metadata. После выполнения DROP TABLE таблица больше не существует.

TRUNCATE полностью очищает таблицу. Удаляются все строки. Структура таблицы сохраняется.

<img width="718" height="288" alt="image" src="https://github.com/user-attachments/assets/63fd43f2-42e3-441d-b453-4e4b8fcdbb81" />

Нарастающий итог (или кумулятивная сумма) 

Это последовательность сумм, где каждая текущая сумма включает все предыдущие значения. 
В SQL для расчёта нарастающего итога используется оконная функция `SUM()` с определением окна через `OVER()`.

Пример: Таблица employees, содержит информацию о сотрудниках. Необходимо вывести общие затраты компании на зарплату сотрудникам нарастающим итогом для каждого отдела.

Запрос:

```
SELECT *, 
SUM(salary) OVER(PARTITION BY department ORDER BY id) AS cumulative_sum
FROM postgres.test.employees
```
**5) Какие будут результаты выполнения следующих SQL-запросов?**

1) MIN(id1)
2) SUM(id1)
3) SUM(id1 + id2)
4) AVG(id1)
5) SUM(8)
6) SUM(id3)

7) COUNT(*)
8) COUNT(1)
9) COUNT(id1)
10) COUNT(id3)
11) COUNT('id1')
12) COUNT(DISTINCT id1)

для следующей таблицы:

```
id1     id2     id3
null    1       null
2       2       null
5       null    null
null    null    null
2       3       null
```

Результаты запросов: 

MIN(id1) = 2. Минимальное значение в столбце id1 (игнорируются NULL значения) — это 2.

SUM(id1) = 9. Суммируются все значения в столбце id1, игнорируя NULL: 2 + 5 + 2 = 9.

SUM(id1 + id2) = 9. Суммируются значения id1 + id2 по каждой строке. Для строк с NULL значения в этих выражениях игнорируются:

```
Строка 1: null + 1 → NULL (игнорируется).
Строка 2: 2 + 2 → 4.
Строка 3: 5 + null → NULL (игнорируется).
Строка 4: null + null → NULL (игнорируется).
Строка 5: 2 + 3 → 5.
Итоговая сумма: 4 + 5 = 9.
```

AVG(id1) = 3. Среднее значение по столбцу id1 (игнорируются NULL значения): (2 + 5 + 2) / 3 = 3.

SUM(8) = 40. В этом случае сумма 8 будет вычисляться для каждой из 5 строк, т.е. 8 * 5 = 40.

SUM(id3) = NULL. Столбец id3 состоит только из NULL значений. Функция SUM() игнорирует NULL и возвращает NULL, так как нет числовых значений для суммирования.

COUNT(*) = 5. COUNT(*) подсчитывает количество всех строк в таблице, включая те, где есть NULL.

COUNT(1) = 5. COUNT(1) подсчитывает количество строк, независимо от значений в столбцах. Результат — количество строк в таблице, т.е. 5.

COUNT(id1) = 3. COUNT(id1) подсчитывает количество строк, где значение в столбце id1 не NULL. В данном случае значения id1 равны 2, 5, и 2 (3 строки).

COUNT(id3) = 0. Столбец id3 состоит только из NULL значений, и поэтому COUNT(id3) возвращает 0, так как все значения NULL не учитываются.

COUNT('id1') = 5. При использовании COUNT('id1') в качестве выражения будет подсчитано количество строк, поскольку строка 'id1' не равна NULL. Таким образом, результат будет 5.

COUNT(DISTINCT id1) = 2. COUNT(DISTINCT id1) подсчитывает уникальные значения в столбце id1. В данном случае уникальные значения — это 2 и 5.

**7) Какое будет максимальное и минимальное количество строк при различных JOIN?**

`INNER`: минимальное - 0 (все значения разные), максимальное - N * N (в первой и во второй таблице все записи одинаковые);

`LEFT`: минимальное - N (все значения разные), максимальное - N * N (в первой и во второй таблице все записи одинаковые);

`RIGHT`: минимальное - N (все значения разные), максимальное - N * N (в первой и во второй таблице все записи одинаковые);

`FULL`: минимальное количество - N (максимум из двух таблиц), максимально - N * N (в первой и во второй таблице все записи одинаковые, то есть левая таблица 1 1 правая таблица 1 1 1 итог 6 строк);

`CROSS`: максимальное и минимальное количество - N*N (так как это декартово произведение);

Таблица левая = 4 строки, таблица правая = 5 строк, сколько минимально и максимально строк при FULL JOIN, привести пример на данных? Что будет если есть NULL, дубли, все значения разные, все значения одинаковые?

Минимально 5, максимально 20 если все значения одинаковые(дубли), если все значения разные то максимум 4 + 5 = 9 строк. Если есть NULL, то NULL не равен NULL, поэтому такие строки не матчятся и попадают как несопоставленные.  

**8) Какие есть оконные функции ?**

`ROW_NUMBER()` — присваивает уникальный порядковый номер строкам внутри окна, без повторов, просто 1, 2, 3…

`RANK()` — присваивает ранг с пропусками при одинаковых значениях: если есть дубликаты, они получают одинаковый ранг, а следующий ранг пропускается.

`DENSE_RANK()` — как `RANK()`, но без пропусков: одинаковые значения получают один ранг, и следующая позиция идёт подряд.

`LAG()` — возвращает значение из предыдущей строки в рамках окна (используется для сравнения с прошлым значением).

`LEAD()` — возвращает значение из следующей строки в рамках окна (используется для сравнения с будущим значением).

**12) Назовите физические виды JOIN и их принцип работы?**

Как именно СУБД выполняет соединение на уровне алгоритмов.

`Nested Loop Join` - Для каждой строки из первой таблицы перебираются строки второй.
